vpnsocksify

Route any application's traffic through a VPN using a lightweight Docker SOCKS5 proxy. Supports OpenVPN and WireGuard.

View on GitHub Quick Start 
Your App  ──SOCKS5──▶  vpnsocksify  ──VPN Tunnel──▶  Internet
(browser, curl)      (Docker container)        (via VPN exit IP)
                     iptables kill switch
                     DNS leak prevention

Features

🔐

Kill Switch

iptables rules block all traffic outside the VPN tunnel. If the VPN drops, nothing leaks.

🔄

Dual VPN Support

Auto-detects OpenVPN (.ovpn) and WireGuard (.conf) configs. Just drop them in and go.

🔒

SOCKS5 Authentication

Optional username/password authentication for the SOCKS5 proxy. RFC 1929 compliant.

📦

Tiny Footprint

Alpine Linux base image. The entire container is ~47MB. Runs anywhere Docker runs.

🔁

Multi-Instance

Run multiple containers on different ports with different VPN configs simultaneously.

🧪

Health Checks

Built-in Docker health check verifies the entire chain: proxy, tunnel, and internet access.

Quick Start

1

Clone and add your VPN config

git clone https://github.com/faeton/vpnsocksify.git
cd vpnsocksify
cp /path/to/your-vpn.ovpn config/
cp .env.example .env
2

Build and run

docker compose build
docker compose up -d
3

Use the proxy

curl --proxy socks5h://proxyuser:proxypass@localhost:1080 https://api.ipify.org
# Returns your VPN exit IP, not your real IP

Configuration

Variable Default Description
SOCKS_PORT1080SOCKS5 proxy listen port
SOCKS_USERemptySOCKS5 username (empty = no auth)
SOCKS_PASSemptySOCKS5 password
VPN_CONFIGemptyConfig filename to use (auto-detected if empty)
VPN_USERemptyVPN username (OpenVPN auth-user-pass)
VPN_PASSemptyVPN password
BIND_ADDR0.0.0.0Bind address (0.0.0.0 = all, 127.0.0.1 = local)
KILL_SWITCHtrueEnable iptables kill switch
DNS_SERVERS1.1.1.1,8.8.8.8DNS servers (comma-separated)
